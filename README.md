# cg-lab-work0：基于Taichi的GPU粒子群交互仿真

## 一、项目简介

本项目基于Python与Taichi实现了一个面向图形计算入门的GPU粒子群交互仿真系统。项目采用 `src` 目录结构组织代码，通过模块化方式划分参数配置、物理计算与渲染显示三部分，并借助Taichi的GPU后端完成大量粒子的并行更新与实时可视化。

程序运行后会创建交互式窗口，窗口中的粒子在鼠标位置作用下产生明显的聚集与跟随行为；当粒子运动到窗口边界时，会发生碰撞与反弹，从而形成连续、流畅的动态粒子效果。该项目体现了GPU并行计算在粒子系统仿真中的基础应用，也展示了模块化代码组织在图形程序开发中的实际价值。

## 三、代码说明

### 1. `__init__.py`
包初始化文件，用于将 `Work0` 目录识别为标准 Python 模块，便于使用模块化方式运行程序。

### 2. `config.py`
参数配置中心，用于统一管理粒子系统的关键参数，包括：
- 粒子总数 `NUM_PARTICLES`
- 鼠标引力强度 `GRAVITY_STRENGTH`
- 空气阻力系数 `DRAG_COEF`
- 边界碰撞反弹系数 `BOUNCE_COEF`
- 窗口分辨率 `WINDOW_RES`
- 粒子半径 `PARTICLE_RADIUS`
- 粒子颜色 `PARTICLE_COLOR`

### 3. `physics.py`
核心物理计算模块，负责粒子系统的 GPU 并行更新，主要实现：
- 粒子位置与速度的数据场定义
- 粒子随机初始化
- 鼠标引力作用
- 速度衰减
- 位置更新
- 边界碰撞检测与反弹处理

### 4. `main.py`
程序入口与渲染模块，负责：
- 初始化 Taichi GPU 环境
- 初始化粒子系统
- 创建图形窗口
- 获取鼠标位置
- 调用物理更新函数
- 将粒子实时绘制到窗口中

## 四、功能说明

本项目实现了以下功能：
- 基于 Taichi GPU 后端进行粒子并行计算
- 实现粒子随机初始化
- 实现鼠标交互吸引效果
- 实现粒子边界碰撞与反弹
- 支持粒子实时渲染显示
- 支持 `src` 结构下的模块化运行

## 五、效果展示

程序运行后会弹出交互窗口，窗口中的粒子会在鼠标位置作用下产生聚集与跟随效果，并在边界处发生碰撞反弹，形成连续的动态粒子运动效果。
